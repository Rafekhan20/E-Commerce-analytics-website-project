# Power BI E-Commerce Analytics Setup Guide

This guide describes how to import your exported e-commerce analytical datasets into Power BI Desktop, build a standard star schema, and implement advanced business intelligence measures using DAX.

---

## 1. Importing the Datasets

1. Open **Power BI Desktop**.
2. Go to **Home** &rarr; **Get Data** &rarr; **Text/CSV**.
3. Navigate to your workspace directory: `power_bi/data/`.
4. Import all 6 CSV files generated by the exporter:
   - `users.csv` (Dim Table)
   - `products.csv` (Dim Table)
   - `orders.csv` (Fact Table)
   - `order_items.csv` (Fact Table)
   - `web_events.csv` (Fact / Log Table)
   - `rfm_analysis.csv` (Dim/Analytical Table)
5. Select **Load** for each dataset.

---

## 2. Building the Star Schema (Model View)

Navigate to the **Model View** (left-hand sidebar) and configure relationships between tables as follows. (Ensure relationships are active and cardinalities are correct):

```
       +-----------------------+              +------------------------+
       |     Dim_Products      |              |       Dim_Users        |
       +-----------------------+              +------------------------+
       | Product_ID (PK)       |              | User_ID (PK)           |
       | Product_Name          |              | Signup_Date            |
       | Category              |              | State, Age, Gender     |
       | Price, Cost, Stock    |              | Acquisition_Channel    |
       +-----------------------+              +------------------------+
                   |                                       |
                   | 1                                     | 1
                   |                                       |
                   | *                                     | *
       +-----------------------+              +------------------------+
       |   Fact_OrderItems     |              |      Fact_Orders       |
       +-----------------------+              +------------------------+
       | Item_ID (PK)          |              | Order_ID (PK)          |
       | Order_ID (FK) --------|------------> | User_ID (FK)           |
       | Product_ID (FK)       |              | Order_Date             |
       | Quantity, Unit_Price  |              | Status, Payment_Method |
       +-----------------------+              +------------------------+
                                                           | 1
                                                           |
                                                           | *
                                              +------------------------+
                                              |     Fact_WebEvents     |
                                              +------------------------+
                                              | Event_ID (PK)          |
                                              | Session_ID, User_ID    |
                                              | Event_Time, Event_Type |
                                              +------------------------+
```

### Key Relationships Setup:
1. Connect `Dim_Products[product_id]` to `Fact_OrderItems[product_id]` (**1-to-many**, Single direction).
2. Connect `Fact_Orders[order_id]` to `Fact_OrderItems[order_id]` (**1-to-many**, Single direction).
3. Connect `Dim_Users[user_id]` to `Fact_Orders[user_id]` (**1-to-many**, Single direction).
4. Connect `Dim_Users[user_id]` to `Fact_WebEvents[user_id]` (**1-to-many**, Single direction - set Cross-filter direction to *Both* if session-level user logs need filtering by user dimensions).

## 3. Essential DAX Measures

Create a new table `_Measures` to house these calculations. Right-click the table and choose **New Measure**:

### Revenue & Profitability Measures

1. **Total Sales (Gross Revenue)**:
   ```dax
   Total Revenue = SUMX(Fact_OrderItems, Fact_OrderItems[quantity] * Fact_OrderItems[unit_price])
   ```

2. **Total Cost of Goods Sold (COGS)**:
   ```dax
   Total Cost = SUMX(Fact_OrderItems, Fact_OrderItems[quantity] * RELATED(Dim_Products[cost]))
   ```

3. **Gross Profit**:
   ```dax
   Gross Profit = [Total Revenue] - [Total Cost]
   ```

4. **Gross Profit Margin (%)**:
   ```dax
   Gross Margin % = DIVIDE([Gross Profit], [Total Revenue], 0)
   ```

5. **Year-to-Date (YTD) Revenue** (Requires a standard Date/Calendar table in Power BI):
   ```dax
   YTD Revenue = TOTALYTD([Total Revenue], 'Calendar'[Date])
   ```

### Transactional Performance

6. **Average Order Value (AOV)**:
   ```dax
   Average Order Value = DIVIDE([Total Revenue], DISTINCTCOUNT(Fact_Orders[order_id]), 0)
   ```

7. **Total Orders Count**:
   ```dax
   Total Orders = DISTINCTCOUNT(Fact_Orders[order_id])
   ```

### Customer & Behavioral Analytics

8. **Web Conversion Rate (%)**:
   Computes the ratio of sessions containing a purchase to total web sessions.
   ```dax
   Session Conversion Rate % = 
   VAR TotalSessions = DISTINCTCOUNT(Fact_WebEvents[session_id])
   VAR PurchaseSessions = CALCULATE(
       DISTINCTCOUNT(Fact_WebEvents[session_id]), 
       Fact_WebEvents[event_type] = "purchase"
   )
   RETURN
   DIVIDE(PurchaseSessions, TotalSessions, 0)
   ```

9. **Customer Lifetime Value (CLV)**:
   Estimates cumulative spend per user.
   ```dax
   Customer Spend Average = DIVIDE([Total Revenue], DISTINCTCOUNT(Fact_Orders[user_id]), 0)
   ```

10. **RFM Customer Grouping Count**:
    Helper to slice user count by the segments exported in `rfm_analysis.csv`:
    ```dax
    Segment Customer Count = CALCULATE(COUNT(rfm_analysis[user_id]))
    ```

## 4. Suggested Dashboard Layout

To create a professional analytics suite, design the following three pages:

### Page 1: Executive Overview (Revenue Analysis)
* **Top Ribbon**: KPI Cards for `Total Revenue`, `Total Orders`, `AOV`, and `Gross Margin %`.
* **Area Chart**: Monthly Revenue vs. Gross Profit trends.
* **Donut Chart**: `Total Revenue` by `Dim_Products[category]`.
* **Slicers**: Date range selector, User State, and Acquisition Channel.

### Page 2: Product Performance
* **Bar Chart**: Top 10 products by units sold (`SUM(Fact_OrderItems[quantity])`).
* **Matrix View**: Row headers: `Category` &rarr; `Product Name`. Values: `Quantity Sold`, `Total Revenue`, `Gross Profit`, `Gross Margin %`.
* **Scatter Plot**: Price (Y-axis) vs. Quantity Sold (X-axis) colored by Category to analyze price sensitivity.

### Page 3: Customer Behavior
* **Funnel Chart**: Count of Session IDs grouped by `event_type` (`page_view`, `product_view`, `add_to_cart`, `purchase`).
* **Column Chart**: `Total Revenue` and `Conversion Rate %` by `Dim_Users[acquisition_channel]`.
* **Treemap**: Customer counts by RFM segments (`rfm_analysis[customer_segment]`).
